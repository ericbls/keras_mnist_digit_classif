Este projeto ir analisar a base de dados MNIST incluso no pacote KERAS, com o objetivo de treinar uma rede neural para melhor avaliar imagens manuscritas de nmeros de 0 a 9..

Aps treinado, ser importado uma base de imagens de manuscritos sem a classificao que permite realizar a validao, para prever quais nmeros cada imagem manuscrita refere-se.

Preparao da base de imagens

# Base de dados utilizada para treinar o modelo

# Separao da base em treino X e treino Y.

# Definio dos valores para validao (x_test e y_test)

Geralmente, para separar a base em treino e teste, utilizamos a funo split em uma proporo determinada, como 70% treino e 30% teste. Mas como a base MNIST j vem com uma separao predeterminada, j podemos definir os dataframes de treino e teste.

# matriz com 60000 imagens e 784 colunas (cada linha = 1 imagem)
x_train <- array_reshape(x_train, c(nrow(x_train), 784))
x_test <- array_reshape(x_test, c(nrow(x_test), 784))

# rescale - retifica os pixels em uma escala binria (preto e branco)
# Antes deste comando, haveria tons de cinza
x_train <- x_train / 255
x_test <- x_test / 255

# Os dados y s?o um vetor inteiro com valores variando de 0 a 9.
#Para preparar esses dados para treinamento, codificamos os vetores
#em matrizes de classe bin?ria usando a fun??o Keras to_categorical():

y_test <- to_categorical(y_test, 10)

Acima realizado a transformao dos dados de imagens em um dataframe de 2 dimenses, em que cada linha representa uma imagem, em formato de lista dos valores dos pixels.

Modelo de Rede Neural 1

Nesta seco o modelo neural treinado o mesmo que o executado no exemplo de aula.

Configurao do modelo 1

# Pr-configurando o modelo de rede neural
modelo1 <- keras_model_sequential() 
modelo1 %>% 
  layer_dense(units = 256, activation = 'relu', input_shape = c(784)) %>% 
  layer_dropout(rate = 0.4) %>% 
  layer_dense(units = 128, activation = 'relu') %>%
  layer_dropout(rate = 0.3) %>%
  layer_dense(units = 10, activation = 'softmax')

O modelo treinado acima tem a seguinte estrutura:

  • layer_dense(units = 256, activation = 'relu', input_shape = c(784)) : Cria uma camada interna com 256 neurnios, que receber 784 valores (todos os pixels de 1 imagem), e utilizar a funo ReLU, como parmetro de ativao.

  • layer_dropout(rate = 0.4) : Para evitar overfittin, 40% das conexes entre os neurnios de entrada e a primeira camada de neurnios intermedirios, so eliminadas aleatroriamente. Isso tambm melhora o desempenho computacional.

  • layer_dense(units = 128, activation = 'relu') : Nesta outra camada intermediria o mesmo realizado que na primeira camada intermediria. Contudo, com 128 neurnios.

  • layer_dropout(rate = 0.3) : desta vez apenas 30% das conexes so eliminadas.

  • layer_dense(units = 10, activation = 'softmax')` : finalmente, camada de Output possui 10 ndulos, cada um representa a chance de uma previso ser um nmero, entre 0 e 9. O valor deste ndulo vai de 0 at 1, graas funo softmax, que garante que a soma de todas as chances d 1.

Modelo de Rede Neural 2

## Defina o formato da entrada
img_rows   <- 28   # Linhas das imagens mnist
img_cols   <- 28   # Colunas das imagens mnist
channels   <- 1    # 1 = escala de cinza, 3 = RGB
num_classes <- 10  # dgitos de 09 (exemplo MNIST)

# Pr-configurando o modelo
modelo_cnn <- keras_model_sequential() %>%
  # 1 camada convolucional
  layer_conv_2d(filters = 32,
                kernel_size = c(3, 3),
                activation  = "relu",
                input_shape = c(img_rows, img_cols, channels)) %>%
  # 2 camada convolucional
  layer_conv_2d(filters = 64,
                kernel_size = c(3, 3),
                activation  = "relu") %>%
  # Pooling
  layer_max_pooling_2d(pool_size = c(3, 3)) %>%
  # Dropout
  layer_dropout(rate = 0.5) %>%
  # Achata para um vetor
  layer_flatten() %>%
  # Densa oculta
  layer_dense(units = 250, activation = "sigmoid") %>%
  # Sada
  layer_dense(units = num_classes, activation = "softmax")

Ao construir a estrutura da rede neural, antes, definimos os parmetros de como a imagem do banco ser recebida no input. Cada imagem tem 28 x 28 pixels, e em geral pode ter at 3 camadas com informaes de cor (RGB). No caso deste laboratrio, estamos utilizando a base MNIST, que contm apenas 1 canal (preto e branco).

Explicando:

A primeira camada layer_conv_2d(filters = 32, kernel_size = c(3, 3), activation = "relu", input_shape = c(img_rows, img_cols, channels))

Um filtro convolucional 2D cria um filtro que varre a imagem em partes. O parmetro kernel_size de 3,3 define que este modelo receber do neurnio de entrada uma imagem 28x28, e ele ir analisar ela em quadrados de 3x3.

O parmetro filters determina o nmero de filtros diferentes que sero aplicados. Desta forma, o nmero de canais que era 1 na entrada, se tornar 32 (uma nova imagem com a aplicao de cada filtro).

Como cada janela do kernel_size resulta em 1 novo valor de pixel, e ao aplicar o filtro, aumentamos o nmero de channels, de certa forma, os outputs dessa camada convolucional ter mais channels , mas em contrapartida, sero imagens com menos pixels, pois no foi definido um padding, que adiciona uma borda nas imagens antes de aplicar os filtros.

A segunda camada de neurnios convolucionais 2d segue o mesmo princpio da primeira, mas com 64 filtros. importante ressaltar que as camadas convolucionais no so densas, isto , no h conexes entre todos os neurnios da camada prvia com a posterior. Em contrapartida, camadas convolucionais de neurnios compartilham entre si os outputs do filtro criado com neurnios diferentes da prxima camada. Isto , uma mesma imagem que foi transformada por um neurnio convolucional 2d gerar vrios outputs, e cada um deles ir para neurnios diferentes da prxima camada.

A terceira camada uma operao de max_pooling, que, neste caso, reduz a imagem aplicando em pedaos menores dela, de 3x3 pixels, um filtro que mantm no output apenas 1 pixel de valor mximo naquela janela.

A quarta camada um dropout de 50% das conexes entre os neurnios, ao realizar o treinamento, que previni overfitting e uma sobrecarga computacional.

A quinta camada realiza o que foi feito no exemplo do laboratrio 4 (ou modelo 1 deste trabalho), em que as imagens so unidas de forma convertida, em um dataframe, cujas linhas representam 1 imagem cada uma, e as colunas possuiro os valores de cada pixel daquela imagem. Isso permite adaptar as informaes processadas at agora como array 4D em uma simples tabela 2D, em que os dados podem ser tratados de forma densa.

A Sexta camada cria 250 neurnios com funo de ativao sigmoid.

Enfim A Stima camada so os neurnios de sada com a funo de ativao softmax para poder criar uma classificao One Hot Encode, e finalmente determinar, para cada imagem do input, qual nmero de 0 at 9 ela .

Em resumo:

Resumo do fluxo:

  1. Entrada: imagem 28x28x1 (preto e branco)

  2. Conv1: detecta bordas e padres simples

  3. Conv2: detecta padres mais complexos

  4. Pooling: reduz tamanho da imagem

  5. Dropout: desativa 50% dos neurnios

  6. Flatten: transforma imagem em vetor

  7. Densa: combina tudo em uma camada com 250 neurnios

  8. Sada: 10 neurnios com softmax (classificao)

O modelo 1 treinado utilizando 30 epochs e um batch size de 128.

O validation split de 0.2.

Importante destacar que antes de treinar este modelo, necessrio alterar a base de treino. Os neurnios convolucionais precisam receber as imagens como um array de 3 dimenses (pixels de X e Y, e channels).

# Ao usar neurnios de convoluo 2D, precisamos que o X_treino volte a ser 4D
x_train_cnn <- array_reshape(x_train, c(nrow(x_train), img_rows, img_cols, channels))

# Compilando o modelo
modelo_cnn <- compile(modelo_cnn,

)

# Parmetros para interromper o treino, caso a acurcia no melhore:
early_stop <- callback_early_stopping(
  monitor = "val_accuracy",    # mtrica que ser monitorada
  patience = 5,                # nmero de pocas sem melhoria at parar
  restore_best_weights = TRUE  # recupera os melhores pesos
)

# Treinando o modelo
history <- modelo_cnn %>%
  fit(x_train_cnn,
      y_train,
      epochs = 30,
      batch_size = 128,
      validation_split = 0.2,
      callbacks = list(early_stop))
